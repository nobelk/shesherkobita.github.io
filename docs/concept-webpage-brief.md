# Shesher Kobita Concept Webpage Brief

## Current Review Status

As of 2026-04-10, this repository contains a working static HTML/CSS
landing page plus the source visual assets. Current review work is
focused on layout refinement, content accuracy, and responsive polish
against the original concept goals documented below.

## Project Goal

Create a ready-to-open static landing page for **Shesher Kobita** that
feels calm, elegant, and nature-connected. The page should combine the
green identity from the logo with a clean white foundation and
restrained black typography.

## Visual Direction

- **Primary palette:** botanical green, white, charcoal/black
- **Mood:** artistic, serene, refined, village luxury with modern comfort
- **Primary inspiration:** `img/shesher_kobita_logo.png`
- **Feature visuals:**
  - `img/shesher_kobita_design1.png`
  - `img/shesher_kobita_design2.png`

## Required Content Sections

The final page should cover all of the following topics in a clear single-page flow:

1. **Concept / Introduction**
   - Position Shesher Kobita as a peaceful retreat concept rooted in
     nature, literature, and reflection.
2. **Facilities**
   - Mention guest rooms, library, courtyard, lounge/veranda-style
     common areas, and quiet shared spaces inspired by the concept
     images.
3. **Village Serenity**
   - Emphasize fresh air, open sky, greenery, and the slower rhythm of village life.
4. **Proximity to Dhaka and Modern Healthcare**
   - Present the location as peaceful yet practical, with access to
     Dhaka and nearby modern healthcare when needed.
5. **Organic Food**
   - Highlight fresh, organic, home-style meals and seasonal ingredients.
6. **Activities**
   - Suggest reading, walking, courtyard relaxation, family time,
     reflection, and low-key cultural or nature-based activities.
7. **Worship Access**
   - State that mosque access is conveniently nearby.
8. **Reservation Contact**
   - Include: `shesherkobita@gmail.com`

## Content Priorities

- Keep copy warm, reassuring, and premium without sounding corporate.
- Favor short paragraphs and readable section groupings.
- Use the two concept design images prominently, not as incidental thumbnails.
- Ensure the reservation contact is easy to find near the end of the page.

## Recommended Information Architecture

1. Header / hero with logo and short tagline
2. Introductory concept statement
3. Featured concept gallery using both provided design images
4. Facilities and experience overview
5. Serenity / location / healthcare / food / activities / worship sections
6. Reservation/contact call to action
7. Footer with concise brand reinforcement

## Responsive Design Expectations

### Mobile

- Single-column layout
- Comfortable tap targets and spacing
- Images scale to full width without clipping key details
- Text remains readable without zooming

### Tablet/Desktop

- Wider hero composition
- Gallery or split layouts may be used where appropriate
- Maintain generous whitespace and balanced alignment
- Avoid over-compressing content into dense multi-column blocks

## Implementation Notes for the Static Page

- Prefer semantic HTML5 sections (`header`, `main`, `section`, `figure`, `footer`).
- Keep the page dependency-light: plain HTML/CSS is sufficient.
- Optimize for direct local opening in a browser without a build step.
- Preserve accessible alt text for logo and concept artwork.
- Use consistent spacing, a restrained type scale, and a small, coherent color system.

## Asset Notes

- `design1` communicates the finished visual mood with color,
  materials, courtyard, and library imagery.
- `design2` communicates concept planning, architectural sketching, and layout intent.
- Using both assets together helps show both aspiration and planning depth.
